# Tableau AI PR Agent Roadmap

## 目的

`tableau-ai-pr-agent` は、Tableau ダッシュボード内で動く **AI広報アクション** のための拡張機能です。

チャットアプリではなく、分析結果をもとに広報投稿を素早く作成し、確認し、必要なら Slack へ投稿することを目的にします。

主な利用シーンは次のとおりです。

- ユーザー会やイベントの事前告知
- 開催直前のリマインド
- 開催中の実況
- 開催後のお礼・レポート
- 次回参加の呼びかけ

## 最終デモストーリー

1. Tableau 拡張機能の画面で「AI広報アクション」を開く
2. 投稿タイプを選ぶ
3. イベント名、TechPlay イベントページ URL、現在の状況などを入力する
4. 必要に応じて会場写真をアップロードする
5. Tableau の分析結果、TechPlay ページ、将来の Drive 資料、会場写真を参照する
6. Agent が Slack 投稿文、画像、ハッシュタグ、根拠、確認ポイントを生成する
7. 画像を S3 に保存し、URL を表示する
8. Slack Incoming Webhook で画像付き投稿を送る

## MVP 範囲

最初のデモ可能な MVP は次の範囲に限定します。

- Phase 1: AI広報アクション UI
- Phase 2: Action Runs API
- Phase 3: TechPlay ページ読み込み
- Phase 4: Tableau MCP で固定分析
- Phase 5: Slack Webhook 投稿
- Phase 6: S3 画像 URL 表示

この MVP では、Google Drive、スマホ写真の本格取り込み、画像生成の高度化は後回しにします。

## Phase 一覧

### Phase 1: AI広報アクション UI

- チャット UI をやめて、広報アクション入力・プレビュー画面を作る
- 投稿タイプ、イベント名、URL、現在の状況などを入力できるようにする
- ダミーの投稿文、画像枠、根拠、確認ポイントを表示する
- まだ API にはつながない

### Phase 2: Action Runs API

- `/action-runs` のような広報アクション用 API を作る
- 既存の非同期ジョブ構造を流用する
- まだ本格的な Tableau MCP 分析や Slack 投稿はしない

### Phase 3: TechPlay ページ読み込み

- TechPlay イベント URL からイベント名、日時、概要を抽出する
- 初期実装は HTTP fetch と HTML 抽出でよい
- Web 検索 MCP や Playwright MCP は後回しにする

### Phase 4: Tableau MCP 分析専用化

- 自由質問ではなく、広報判断に必要な固定分析を行う
- 投稿タイプ別、キーワード別、曜日・時間帯別、画像有無などの傾向を見る
- `query-datasource` / `get-datasource-metadata` を中心に使う

### Phase 5: Slack Webhook 投稿

- Slack Incoming Webhook で投稿文、根拠、確認ポイントを送る
- 画像は S3 URL を image block として表示する

### Phase 6: S3 画像保存基盤

- 生成画像を保存する S3 バケットを用意する
- Slack から読める URL を生成する
- 90 日後に削除する Lifecycle を設定する
- ファイル名は UUID など推測困難なものにする

### Phase 7: 画像生成

- 最初は完全な画像生成 API より、テンプレート画像生成でもよい
- 白、淡い水色、濃紺を基調にする
- イベント名、投稿タイプ、キーワードを画像に入れる

### Phase 8: スマホ写真アップロード

- 会場写真をアップロードできるようにする
- 写真の用途を選べるようにする
- 例: 雰囲気把握だけ、投稿画像の背景に使う
- 人物の写り込みに注意し、完全自動投稿はしない

### Phase 9: Google Drive 資料参照

- 最終的には Google Drive 資料を参照したい
- ただし OAuth と Drive API 設定が重いので後回しにする
- 初期版では S3 や固定 Markdown で代替してよい

## 後回しにするもの

当面は次を後回しにします。

- Google Drive の本格連携
- スマホ写真の高度な解析
- 本格的な画像生成 API
- Web 検索 MCP や Playwright MCP の導入
- 自由質問型の汎用チャット体験
- 既存チャット UI の即時削除
- Notion 連携の即時削除

## 既存アプリから流用するもの

既存の `tableau-chat-extension` 相当の基盤から、次を壊さず流用します。

- Tableau Extension SDK 初期化
- Cognito 認証
- API Gateway + Lambda
- Tableau MCP 接続
- GitHub Actions デプロイ骨格
- CloudFormation の基本構成
- 非同期ジョブ管理パターン
- AWS 共通ユーティリティ
- ログ出力
- セキュリティ関連ユーティリティ

## 今後置き換えるもの

段階的に次を置き換えます。

- チャット中心の UI
- 自由質問中心のプロンプト設計
- Notion 連携の既定フロー
- 汎用的な回答生成ロジック

置き換えは一気に削除せず、広報アクションの導線が安定してから進めます。

## 主要コンポーネントの位置づけ

### Slack

- 最終出力先
- 画像付きの広報投稿を送る
- 投稿文、根拠、確認ポイントを含める

### S3

- 画像の一時保管先
- Slack から参照できる公開 URL を提供する
- 90 日で消える前提の保管にする

### 画像

- 最初はダミー枠から始める
- 後で生成画像、テンプレート画像、会場写真の扱いを追加する
- 投稿内容の補助情報として使う

### TechPlay

- イベントの公式情報ソース
- イベント名、日時、概要を抽出する
- 広報文の土台情報として使う

### Tableau MCP

- 過去 1 年の X 投稿データ分析に使う
- 広報判断に必要な固定分析を行う
- 自由質問の汎用チャットではなく、広報意思決定の材料を返す

## 実装の進め方

- 1 回の作業では 1 Phase だけ進める
- 既存アプリを壊さない
- 破壊的変更は事前に提案する
- まずは MVP 範囲を安定させ、その後に後続 Phase を追加する
